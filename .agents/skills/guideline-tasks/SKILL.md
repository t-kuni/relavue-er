# タスク一覧(TASK.md)作成ガイドライン

* タスク毎にチェックボックスを付ける
* 遂行は他者に任せるので可能な限り詳細に記載する
    * コードを大量に記載するのはNG
    * 参照するべき仕様書(spec)がある場合は仕様書へのリンクを書く
* 編集対象のファイルを明確にする
    * 変更点の概要を記載する
    * APIや関数を追加する場合はインタフェースを明確にする
* 「〇〇は不要」や「◯◯のタスクは削除」など指示した場合は、不要である旨を明示的に記載する。
* 本タスクより先に実装・リファクタリングしたほうが良いものの候補があれば「事前修正提案」として記載する
* 連番は付与しない（修正コストが高くなるため）
* 工数は記載しない
