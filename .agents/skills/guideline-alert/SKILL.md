# 遂行障害リスト(ALERT.md)作成ガイドライン

* 作業を遂行する上で障害となる問題(※)が発生した場合、ALERT.mdに記載する
  * 先に既存ファイルを削除する。`rm -f ALERT.md`
  * ※仕様の矛盾、仕様の検討不足、懸念事項、タスクの不備、意思決定を仰ぎたい、など
* 後述のテンプレートに従って記載する
* `[分類]` には以下の分類を記載する
    - 要追加調査問題: このリポジトリ内を調査すれば解決する問題
    - 要リサーチ問題: リポジトリ外の情報を参照しないと解決できない問題
    - 要意思決定問題: 指示者の意思決定が必要な問題
* 対策案がある場合は記載してください。
* `[回答]` は指示者が回答を記載するので記入不要。
* ALERT.mdを作成した場合は、作業の最後に、指示者に「⚠ ALERT.mdを作成したので回答をお願いします」と分かりやすく伝えてください
    * ALERT.mdに記載した内容を指示者に伝える必要はありません（ALERT.mdを見ればわかるので。）

```markdown
# 🟦 1. [概要1]

分類： [分類1]

[本文1]

対策案A: xxx
対策案B: xxx
対策案C: xxx

### 🟠 回答

* [回答1]

# 🟦 2. [概要2]

分類： [分類2]

[本文2]

対策案A: xxx
対策案B: xxx
対策案C: xxx

### 🟠 回答

* [回答2]

...(以下繰り返し)

```
